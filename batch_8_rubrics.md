## 1. CORE-ECON-03

rubric_id: CORE-RUBRIC-B08-CORE-ECON-03
task_type: Core
research_object: generative-AI system inventory controls
answer_unit: inventory exemption rule and record-retention practice

### Key Facts From Source Bundle

- Organisations should define inventory exemptions for generative-AI systems embedded into application software.
- Organisations should maintain a document-retention policy.
- Retention covers test, evaluation, validation, verification, and digital-content transparency records.
- Inventory exemptions are not absence of inventory or governance.
- Embedded application components are distinct from standalone systems.
- Document retention is distinct from incident disclosure.
- Framework controls are not automatically binding legal duties.

### Expected Response Sections

- Conclusion stating the exemption rule and retention practice.
- Source comparison anchored in the controlling framework source.
- Boundary explanation about exemptions, embedded systems, retention, and legal status.
- Caveat about framework status and implementation context.
- Confidence level.

### Fixed Criteria, 0-2 Each

**Coverage**
- 2: Includes exemption rule, retention policy, retained record types, evidence, caveat, and confidence.
- 1: Includes one main control fully and the other partially.
- 0: Does not answer inventory/retention controls.

**Accuracy**
- 2: Correctly states embedded generative-AI inventory exemptions and retention for testing/evaluation/validation/verification/transparency records.
- 1: Gives the two controls but omits some retained record types or embedded-system scope.
- 0: Says embedded systems need no governance or treats retention as disclosure.

**Reasoning**
- 2: Distinguishes exemptions/inventory absence, embedded/standalone systems, retention/disclosure, and framework/legal duty.
- 1: Mentions one boundary but leaves control status unclear.
- 0: Treats framework controls as binding law or removes embedded systems from governance.

**Use of Evidence**
- 2: Uses the controlling framework source and corroborates only compatible framework interpretation.
- 1: Uses one relevant source with limited comparison.
- 0: Uses generic AI governance claims.

**Clarity and Structure**
- 2: Clear two-part control answer with caveat and confidence.
- 1: Mostly clear but incomplete.
- 0: Confusing or generic AI control response.

### Common Failure Modes

- Saying embedded generative-AI systems need not be governed at all.
- Treating document retention as public disclosure.
- Omitting either inventory exemptions or record retention.
- Presenting a framework control as a legal requirement without support.

### Borderline Cases

- Legal comparisons are acceptable only if framework status remains explicit.
- Do not require a full inventory taxonomy.
